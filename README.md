# Escape the Equation：教室危機

Escape the Equation：教室危機 是一款使用 HTML、CSS、JavaScript 製作的瀏覽器數學密室逃脫遊戲。

玩家醒來後發現自己被困在一間神祕教室中。黑板、書桌、時鐘、地圖與終端機都被數學謎題鎖住。玩家必須在倒數時間結束前解開五道隨機謎題，收集密碼碎片，並輸入最終逃脫密碼，才能離開教室。

https://chiou-jiakang.github.io/Escape-the-equation/

## 專案簡介

本專案將數學題目包裝成密室逃脫遊戲機關，而不是單純做成一般選擇題或問答題。

遊戲中的題目包含方程式、比例、百分比、數列規律、座標幾何、二次方程式與函數運算等內容。每一個題型都對應到教室中的不同物件，讓玩家透過點擊物件、解開謎題、取得碎片的方式推進遊戲。

本專案目前已加入隨機題庫系統。每次開始遊戲時，系統會從五個答題區各自的題庫中隨機抽出一題，因此每一局的題目與最終密碼都會不同，避免玩家單純背答案通關。

此外，本專案也加入結局統計與玩家表現評級系統。遊戲結束後，系統會根據玩家的剩餘生命、剩餘時間、提示使用次數與通關狀況，給予不同的逃脫評級，讓遊戲結果不只是「成功」或「失敗」，而是能更具體反映玩家的表現。

## 遊戲玩法

玩家需要調查教室中的五個物件，並分別解開對應的數學謎題。

### 1. 黑板

主要題型為代數與方程式。

玩家需要根據黑板上的未知數關係，推算出密碼片段。這個區域主要測試基本代數運算、變數替換與簡單方程式推理。

### 2. 上鎖的書桌

主要題型為比例、百分比、折扣、濃度與分配問題。

玩家需要根據紙條或配方提示，算出正確數值。這個區域將生活化的比例與百分比問題包裝成抽屜密碼或配方謎題。

### 3. 破鐘

主要題型為數列與規律判斷。

玩家需要觀察數字之間的變化，找出下一個數字。這個區域主要測試玩家對差分、倍數、平方數、立方數與遞增規律的辨識能力。

### 4. 座標地圖

主要題型為座標幾何。

內容包含距離公式、中點、斜率與座標差等概念。玩家需要根據地圖上的座標線索，推算逃生路線或地圖密碼。

### 5. 方程式終端

主要題型為二次方程式、根與係數、判別式與函數輸出值。
玩家需要依照終端機指令計算 access key。這個區域模擬終端機驗證系統，讓數學題目看起來更像一道系統鎖。
解開五個物件後，出口門會解鎖。玩家必須依照出口門規則排列碎片：時鐘 → 地圖 → 終端 → 黑板 → 書桌
由於每局抽到的題目不同，因此每局的最終密碼也會不同。

## 隨機題庫設計

本專案的題庫設計採用「分區題庫」方式，而不是所有題目混在一起抽。

五個答題區各自擁有 15 題題目：

```text
黑板：15 題
書桌：15 題
破鐘：15 題
座標地圖：15 題
方程式終端：15 題
```

每次開始遊戲時，系統會從每個答題區各自隨機抽出一題，因此每局總共會出現五道題目。

### 設計目的

此設計主要有三個目的：

1. 避免玩家背固定答案通關  
   如果題目與最終密碼固定，玩家只要記住答案就可以快速過關，遊戲的解謎性會大幅下降。

2. 維持每個教室物件的數學主題  
   每個物件對應不同類型的數學能力。例如黑板對應代數，破鐘對應數列，座標地圖對應幾何。這樣能讓場景與題型更有一致性。

3. 增加遊戲重玩性與變化度  
   目前共有 75 題候選題，單局會抽出其中 5 題。不同題目組合會產生不同的碎片與最終密碼，使玩家每次遊玩都需要重新解題。

### 抽題流程

遊戲開始時，程式會執行隨機抽題邏輯：

1. 從黑板題庫抽出 1 題
2. 從書桌題庫抽出 1 題
3. 從破鐘題庫抽出 1 題
4. 從座標地圖題庫抽出 1 題
5. 從方程式終端題庫抽出 1 題
6. 將五題的答案分別作為當局密碼碎片
7. 根據出口門規則組合出當局最終密碼

這樣玩家每局都會遇到新的題目組合，不能只靠記憶固定答案通關。

## 動態最終密碼設計

本專案的最終密碼不是固定值，而是根據當局抽到的題目答案自動生成。

玩家每解開一道謎題，就會取得一個密碼碎片。當五個碎片全部收集完成後，出口門會解鎖。玩家需要依照出口門規則排列碎片：

```text
時鐘 → 地圖 → 終端 → 黑板 → 書桌
```

### 生成規則

假設某一局取得的碎片為：
時鐘片段：36
地圖片段：13
終端片段：7
黑板片段：125
書桌片段：600

依照出口門規則：時鐘 → 地圖 → 終端 → 黑板 → 書桌

該局最終密碼會是：36137125600

下一局如果抽到不同題目，碎片與最終密碼也會跟著改變。

### 設計目的

動態密碼設計可以避免玩家單純背誦固定最終密碼，也讓前面的五道謎題真正影響最後的通關結果。玩家必須解出每一道題目並正確排列碎片，才能完成逃脫。

## 輸入防呆設計

為了避免玩家因輸入格式不同而造成錯誤判斷，本專案加入了基本輸入驗證機制。

### 支援輸入

目前支援以下輸入形式：

- 可接受半形數字，例如 `5`
- 可接受全形數字，例如 `５`
- 會自動移除多餘空白，例如 `2 6 4`
- 可接受類似 `x=5` 的輸入格式

### 不接受輸入

以下輸入會被判定為格式錯誤，但不會扣生命：

- 空白輸入
- 中文數字
- 英文字母
- 特殊符號
- emoji
- 非數字形式的完整句子

### 扣分與扣生命規則

輸入錯誤分成兩種情況：

1. 格式錯誤  
   例如輸入中文、英文、特殊符號或空白。此時系統會顯示格式提醒，不會扣生命、不會扣時間，也不會扣分。

2. 格式正確但答案錯誤  
   例如答案應為 `36`，但玩家輸入 `35`。此時系統會判定為真正答錯，並扣生命、扣分與扣時間。

這樣設計可以避免玩家因輸入法或格式問題被不合理懲罰。

### 重複操作防呆

本專案也處理以下重複操作問題：

- 答對後會停用輸入框與按鈕，避免重複提交刷分
- 同一題重複使用提示時，不會重複扣分與扣時間
- 已解開的謎題再次點擊時，只會顯示已取得的碎片，不會再次加分

## 玩家表現評級

遊戲結束後，系統會根據玩家的通關狀況給予表現評級。評級依據包含是否成功逃脫、剩餘生命、剩餘時間與提示使用次數。

目前生命值上限為 3，因此評級標準以三顆愛心作為基準。

### 評級依據

系統主要參考以下指標：

- 是否成功逃脫
- 剩餘生命值
- 剩餘時間
- 使用提示次數
- 是否接近失敗條件

其中，剩餘生命代表玩家的解題穩定度，剩餘時間代表通關效率，提示次數則反映玩家對系統輔助的依賴程度。

### Failed｜逃脫失敗

判定條件：

- 生命值歸零
- 或倒數時間歸零

只要符合其中一項，系統就會判定為逃脫失敗。

### Perfect Escape｜完美逃脫

判定條件：

- 成功逃脫
- 生命值維持 3 / 3
- 使用提示次數為 0
- 剩餘時間至少 300 秒

此評級代表玩家在無傷、無提示，且時間充裕的情況下完成逃脫。由於遊戲總時間為 600 秒，因此剩餘 300 秒以上代表玩家在 5 分鐘內完成所有謎題，屬於最高標準。

### Clean Escape｜俐落逃脫

判定條件：

- 成功逃脫
- 生命值維持 3 / 3
- 使用提示次數不超過 1 次
- 剩餘時間至少 180 秒
- 未達 Perfect Escape 標準
- 未符合 Narrow Escape 條件

此評級代表玩家整體表現穩定。雖然可能使用一次提示或花費較多時間，但仍保持完整生命值，並在合理時間內完成逃脫。

### Narrow Escape｜驚險逃脫

判定條件：

成功逃脫，但符合以下任一條件：

- 生命值只剩 1
- 剩餘時間少於 90 秒
- 使用提示次數達 3 次以上

此評級代表玩家雖然成功逃脫，但過程接近失敗邊緣。可能是生命值過低、時間壓線，或過度依賴提示。

### Escaped｜普通逃脫

判定條件：

- 成功逃脫
- 但不符合 Perfect Escape、Clean Escape 或 Narrow Escape 的條件

此評級屬於成功逃脫後的剩餘分類，代表玩家順利完成遊戲，但整體表現未達高評級，也未嚴重壓線。

### 評級判斷順序

為了避免不同評級條件之間產生模糊判斷，系統會依照以下順序進行判定：

1. Failed｜逃脫失敗
2. Perfect Escape｜完美逃脫
3. Narrow Escape｜驚險逃脫
4. Clean Escape｜俐落逃脫
5. Escaped｜普通逃脫

其中 Escaped 是最後的剩餘分類。也就是說，只要玩家成功逃脫，但不符合 Perfect、Clean 或 Narrow 的條件，就會被歸類為 Escaped。

### 評級設計補充

評級標準的設計並非只看分數，而是綜合考量遊戲過程。

例如：

- 玩家沒有掉生命、沒有使用提示，且快速完成，會得到 Perfect Escape
- 玩家雖然成功通關，但只剩一顆生命，會被判定為 Narrow Escape
- 玩家成功通關，但表現介於高評級與壓線之間，會被判定為 Escaped

這樣可以讓結局更細緻，也能增加玩家再次挑戰更高評級的動機。

## 結局統計

遊戲結束後，結局畫面會顯示以下資訊：

- 玩家評級
- 最終分數
- 剩餘時間
- 剩餘生命
- 答錯次數
- 使用提示次數
- 完成碎片數

### 設計目的

結局統計的目的，是讓玩家能更清楚理解自己的遊戲表現，而不只是單純看到成功或失敗訊息。

玩家可以透過結局統計知道自己：

- 是否花太多時間
- 是否太依賴提示
- 是否答錯太多次
- 是否仍有機會挑戰更高評級

這樣可以提升遊戲的重玩動機，也讓專案更接近完整的遊戲體驗。
