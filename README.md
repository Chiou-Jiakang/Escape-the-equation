# Escape the Equation：教室危機

Escape the Equation：教室危機 是一款使用 HTML、CSS、JavaScript 製作的瀏覽器數學密室逃脫遊戲。  
玩家醒來後，發現自己被困在一間神祕教室中。黑板、書桌、時鐘、地圖與終端機都被數學謎題鎖住。玩家必須在倒數時間結束前，解開五道隨機謎題，收集密碼碎片，並輸入最終逃脫密碼，才能離開教室。

## 專案簡介

本專案將數學題目包裝成密室逃脫遊戲機關，而不是單純做成一般選擇題或問答題。

玩家需要在教室中調查不同物件，解開對應的數學謎題。每解開一道謎題，就會取得一個密碼碎片。當五個碎片全部收集完成後，出口門會解鎖，玩家必須依照指定順序排列碎片並輸入最終密碼。

遊戲目標是在 10 分鐘內成功逃脫，並盡可能減少答錯次數與提示使用次數，以獲得更高的通關評級。

## 遊戲規則

玩家一開始會進入一間被數學謎題控制的教室。

教室中有五個主要謎題物件：

- 黑板
- 上鎖的書桌
- 破鐘
- 座標地圖
- 方程式終端

每個物件都會出現一道隨機數學謎題。玩家答對後，會取得一個密碼碎片。

當五個碎片全部收集完成後：

- 出口門會解鎖
- 玩家可輸入最終密碼

最終密碼排列順序為：

```text
時鐘 → 地圖 → 終端 → 黑板 → 書桌
```

若輸入正確密碼，玩家即可成功逃脫。  
若時間歸零或生命值歸零，則遊戲失敗。

---

## 遊戲初始設定

- 生命值：3
- 初始分數：100
- 倒數時間：600 秒（10 分鐘）

---

## 答題規則

### 答對謎題

玩家答對一般謎題時：

- 分數 +20
- 取得對應密碼碎片
- 播放答對音效

### 答錯謎題

玩家輸入格式正確但答案錯誤時：

- 生命值 -1
- 分數 -10
- 剩餘時間 -15 秒
- 答錯次數 +1
- 播放答錯音效

---

## 提示系統

玩家第一次使用某題提示時：

- 分數 -15
- 剩餘時間 -20 秒
- 提示使用次數 +1

同一題重複查看提示時，不會再次扣分或扣時間。

---

## 輸入防呆設計

本專案將「格式錯誤」與「答案錯誤」分開處理。

如果玩家輸入格式錯誤，系統只會提示玩家修正輸入，不會扣生命、不會扣分，也不會扣時間。

### 會被視為格式錯誤的輸入

- 空白輸入
- 中文數字
- 英文字母
- 特殊符號
- emoji
- 非數字句子

### 系統可接受或轉換的輸入格式

- 半形數字
- 全形數字
- 多餘空白
- x=答案

例如：

```text
５ → 5
x=5 → 5
```

這樣設計的目的，是避免玩家因輸入法或格式問題受到不合理懲罰。

## 隨機題庫設計

本專案採用分區隨機題庫。

目前五個答題區各有 15 題，總共 75 題。每次開始遊戲時，系統會從每個答題區各抽出 1 題，因此每局會出現 5 道謎題。

### 題庫分區

- 黑板：代數與方程式
- 上鎖的書桌：比例、百分比、折扣、濃度與生活應用
- 破鐘：數列與規律
- 座標地圖：座標幾何
- 方程式終端：二次方程式、函數與 access key 題

這樣可以讓每個教室物件都有自己的題型特色，而不是把所有題目混在一起亂抽。

---

## 動態最終密碼設計

本專案的最終逃脫密碼不是固定值，而是由當局取得的五個碎片組合而成。

每道謎題的答案會成為該局的密碼碎片。當玩家收集完五個碎片後，系統會根據固定順序產生最終密碼。

最終密碼順序：

```text
時鐘 → 地圖 → 終端 → 黑板 → 書桌
```

例如某局取得的碎片：

```text
時鐘片段：36
地圖片段：13
終端片段：7
黑板片段：125
書桌片段：600
```

則最終密碼為：

```text
36137125600
```

這樣可以避免玩家背固定答案通關，也提升遊戲重玩性。

## 音效系統

本專案在 Version 0.4.0 加入音效系統，用於提升遊戲互動回饋與密室氛圍。

目前音效包含：

- correct.mp3：玩家答對謎題時播放
- wrong.mp3：玩家答錯謎題時播放
- unlock.mp3：出口門解鎖時播放
- success.mp3：成功逃脫時播放
- gameover.mp3：遊戲失敗時播放

玩家可以透過遊戲畫面右上方的：

```text

```

控制音效是否播放。

音效檔案統一存放於：

```text
assets/sounds/
```

音效來源與用途記錄於：

```text
assets/sounds/README.md
```

## 玩家表現評級

遊戲結束後，系統會根據玩家表現給予不同評級。

評級依據包含：

- 是否成功逃脫
- 剩餘生命值
- 剩餘時間
- 提示使用次數

### Failed｜逃脫失敗

條件：

- 生命值歸零
- 或時間歸零

只要符合其中之一，即判定為失敗。

### Perfect Escape｜完美逃脫

條件：

- 成功逃脫
- 生命值為 3 / 3
- 使用提示 0 次
- 剩餘時間至少 300 秒

代表玩家無傷、無提示，並在 5 分鐘內成功逃脫。

### Clean Escape｜俐落逃脫

條件：

- 成功逃脫
- 生命值為 3 / 3
- 使用提示不超過 1 次
- 剩餘時間至少 180 秒
- 未達 Perfect Escape
- 未符合 Narrow Escape

代表玩家表現穩定，但未達完美標準。

### Narrow Escape｜驚險逃脫

符合以下任一項：

- 成功逃脫，但生命值只剩 1
- 成功逃脫，但剩餘時間少於 90 秒
- 成功逃脫，但使用提示達 3 次以上

代表玩家雖然成功逃脫，但過程非常接近失敗。

### Escaped｜普通逃脫

條件：

- 成功逃脫
- 但不符合 Perfect Escape、Clean Escape 或 Narrow Escape

代表玩家成功逃出教室，但整體表現屬於一般通關。

---

## 評級判斷順序

系統依照以下順序判斷：

1. Failed
2. Perfect Escape
3. Narrow Escape
4. Clean Escape
5. Escaped

這樣可以避免評級條件互相衝突。
